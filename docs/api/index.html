build

Build related operations

Submit a task to build a new C2 implant payload

This endpoint accepts a JSON configuration defining the implant's properties, including its name, output format, and a dictionary of listeners it should communicate with.

The build process is asynchronous. This endpoint returns immediately with a build_uuid which can be used to poll the status via GET /builds/{build_uuid}.

Authorizations:
Bearer_Auth
Request Body schema: application/json
required
implant_name
required
string

Name of the implant

output_format
required
string

Output format (e.g., exe, bin)

listener_uuids
required
Array of strings

The list of listener UUIDs that have their profiles compiled into the implant

initial_get_profile_listener_uuid
required
string

UUID of listener for GET profile

initial_post_profile_listener_uuid
required
string

UUID of listener for POST profile

object

Optional build configuration flags

Responses

Get the status of a build job

Get the status of a specific build job. Contains all of the information about a build, except for payload bytes, and zip bytes.

If you are looking for the payload/source code, Please use:

  • GET /api/v1/build/{payload_hash} to get the payload as a file (bytes)
  • GET /api/v1/build/{payload_hash}/source to get the source code zip
Authorizations:
Bearer_Auth
