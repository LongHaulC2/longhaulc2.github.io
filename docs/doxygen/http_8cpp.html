Executes an HTTP request (typically GET) using WinInet.

Makes a HTTP_GET style request to the server.

  • Parameters
    callback_hostThe target hostname or IP address (e.g., L"192.168.1.10" or L"example.com").
    callback_portThe target port number (e.g., 80 or 443).
    http_verbThe HTTP verb to use (e.g., L"GET").
    uriThe resource path/URI (e.g., L"/index.html").
    headersA vector of formatted HTTP header strings to append to the request.
    request_bodyThe body of the HTTP request (typically empty for GET, but supported here for flexibility).
    responseA reference to a string where the server's returned response body will be appended.
    Returns
    true If the complete connection, request, and response reading sequence succeeds.
    false If any stage of the connection or request fails.

reload: always gets new, no cache, doesn't cache. cache = bad cuz the data is somewhere else besides beacon. Increases detection likelyhood + potential repeat commands
