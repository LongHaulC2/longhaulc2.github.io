22void XOR(unsigned char* data, size_t data_size, const char* key, size_t key_size) {
23 DEBUG_LOG("[XOR] Processing buffer of size: " + std::to_string(data_size) + " with key size: " + std::to_string(key_size));
24 for (size_t i = 0; i < data_size; i++) {
25 /*
26 int division is a bit weird. Note here so I don't have to re-figure this out again later.
27
28 key = 1, key_len = 4. 1/4 is normally .25. an int can't have a decimal.
29 That means 0/4 is 0, with 1 left over (again, 1/4 is invalid, as we can't make a whole number out of it,
30 so 1 is the remainder that does not fit in the whole number)
31
32 Ex, box of pizza. Need 4 pieces to fill a box, and a box cannot be sold without 4 pieces.
33 if you make 1, or 5 pieces, you can't make a box, so you have one "left over".
34
35 That's what this does below.
36
37 first loop would be:
38 data[0] = data[0] ^ key [0 % 4] (which is 0, no remainder left over)
39 data[1] = data[1] ^ key [1 % 4] (which is 1, the remainder)
40 data[2] = data[2] ^ key [2 % 4] (which is 2, the remainder)
41 data[3] = data[3] ^ key [3 % 4] (which is 3, the remainder)
42 data[4] = data[4] ^ key [4 % 4] (which is 4/4, aka 1, the remainer is 0)
43 data[5] = data[5] ^ key [5 % 4] (which is 5/4, the remainer is 1)
44 ... and so on
45 */
46
47 data[i] = data[i] ^ key[i % key_size];
48 }
49}
